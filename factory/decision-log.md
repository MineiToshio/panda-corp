## 2026-06-21 ‚Äî Adaptive repair (patch-in-place before rebuild) + adaptive model escalation (DR-073) ‚Äî plugin v8.34.0, OVERLAY 8.34.0
**What:** Two engine changes that make a localized gate reject CONVERGE cheaply instead of churning, and put the right model on the right work order. **(1) Patch-in-place before rebuild.** When the FRD gate rejects a SPECIFIC work order for a localized CORRECTION fault, the build is typically ~correct except a bounded fault ‚Äî so the engine now PATCHES it in place BEFORE reverting, and revert-and-rebuild (DR-070) becomes the *fallback*. The reviewer no longer reverts on a localized reject: it leaves the WO `IN_REVIEW`, keeps the non-progress cap, and returns **`findings: [{ wo, finding (file:line), failingTest (a RED-proven test), files }]`**. The engine runs `attemptPatch` (opus, `xhigh`): fix ONLY the named finding on the EXISTING build, make the RED test pass, then **re-gate WHOLE-PROJECT** ‚Äî the full FRD adversarial+integration pass AND a whole-project `knip`+`biome`+`tsc` (NOT `--since`: a dead export the patch leaves must not slip to a sibling FRD's global gate ‚Äî the red-team-A fix) ‚Äî committing + `VERIFIED` + `reopen_count:0` only on whole-project-clean. The patch runs SYNCHRONOUSLY inside the gate step, so a sibling never sees broken committed code (DR-070's true invariant preserved without a revert). Only if the patch can't green it whole-project does `revertAndReopen` run (set `PLANNED`, INCREMENT `reopen_count`, surgical `git checkout last_green_sha -- <files>` + `git rm`, never a hard reset). **SINGLE counter:** `reopen_count` is the only budget ‚Äî the in-place patch is a sub-step of one reject cycle, NOT a second `patch_attempts` axis (the CRITICAL refinement over the proposal: it eliminates the nesting/non-termination the red team flagged). **(2) Adaptive model escalation.** `pickWorkerModel(wo)` floors at the mode's `P.worker` and escalates to opus when `difficulty: high` (HYBRID a-priori ‚Äî owner decision) OR `reopen_count >= 1` (empirical ‚Äî it already failed once); the patch always runs on opus. **Cost-weighted brake (red-team-B):** `maxAgents` counts agents not tokens, so `agentSpawned` is weighted by model cost (`COST`, opus‚âà3) ‚Äî an opus escalation can't silently blow the budget; escalations are logged. **(3) HYBRID difficulty field** ‚Äî the WO frontmatter carries `difficulty: low|medium|high` (default medium, backward-compatible), populated by the `work-orders` skill + read by the plan agent; a-priori only for `high` (the one case escalating before the first failure pays), with `reopen_count` as the empirical correction.
**Why:** MC Phase-2's WO-07-005 (the complex Configuraci√≥n surface) needed **4 gate cycles**, each a DIFFERENT progressively-smaller finding (5 EARS ‚Üí reverse cross-nav ‚Üí dead-code knip ‚Üí dead Back button); every reject discarded a ~99%-correct build and the fresh full rebuild re-introduced a new micro-bug. It converged only when a HUMAN recovered the good build from git and patched the one bug (a 4-line fix + a test, ~5 min vs 4 multi-agent passes). Owner: "no tiene sentido descartar todo por un 1% que falla." Revert-and-rebuild is correct only for its original purpose (protecting parallel siblings' global gate); for a localized fault it is waste that actively causes non-convergence. Separately, a complex surface got a sonnet implementer regardless of difficulty or failure history ‚Äî the engine should escalate to opus when a WO is genuinely hard or has failed, without leaving the mode. Red-team-hardened (3 adversarial opus agents): the core idea is sound, v1 had a fatal whole-project-vs-focused re-gate hole (A) and an unbounded-cost/non-termination hole (B) ‚Äî both fixed (whole-project re-gate; cost-weighted brake; one unified budget).
**Impact:** `plugin/templates/shared/.claude/workflows/pandacorp-build.js` (`attemptPatch`, `revertAndReopen`, `pickWorkerModel`, `COST`, `findings` in `FRD_GATE_SCHEMA`, `difficulty`+`reopen_count` in `PLAN_SCHEMA` + the plan prompt, the patch-first reject handler ‚Äî syntax-checked via async-wrap), `plugin/agents/reviewer.md` (report findings + a RED test, don't revert on a localized reject), `plugin/skills/work-orders/SKILL.md` (the `difficulty` rubric), `plugin/skills/implement/SKILL.md` (patch-first + escalation note), `factory/standards/build-orchestration.md` ¬ß6 (gate-reject patch-first + model selection) + ¬ß9 (single budget), `factory/decisions/registry.yaml` (DR-073), `plugin/.claude-plugin/plugin.json` v8.34.0 (MINOR ‚Äî new capability), `plugin/templates/OVERLAY_VERSION` 8.33.0‚Üí8.34.0 (the engine ‚Äî an overlay file ‚Äî changed) ‚Üí MC needs `/upgrade` (its `/implement` preflight does it).
