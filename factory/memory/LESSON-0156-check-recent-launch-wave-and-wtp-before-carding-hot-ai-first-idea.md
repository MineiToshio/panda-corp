**Situation:** a discovery deep-dive on "AI vibe-coding security auditor" — an idea that reads as
AI-first + real pain — found on closer research that it was already saturated: 15-20 near-identical
products had launched within 4-6 months (a fast indie me-too wave, not just slow enterprise
incumbents), a dev.to "I tested every X scanner" shootout already existed comparing them, the platform
itself (Lovable) was eating the wedge for free, and actual willingness-to-pay was measured at only $58
MRR (a vitamin, not a painkiller) via TrustMRR/Microns.

**Lesson:** "AI-first + real pain" is not sufficient signal for a currently-hot category — the entry
cost for an AI-wrapper idea is often a weekend, so a hot category is usually ALREADY crowded with fast
indie clones by the time it looks obviously good, distinct from and faster-moving than the slower
enterprise-incumbent competition a naive competitive scan checks for.

**Apply next time:** before carding an AI-first idea in a currently-hot category, explicitly search
Product Hunt / Show HN / "best `<X>` alternatives" / dev.to-style shootouts for a RECENT launch wave
(weeks-old, not years-old competitors), and check real willingness-to-pay (TrustMRR/Microns or
equivalent) BEFORE treating "AI-first + real pain" as sufficient to proceed.
