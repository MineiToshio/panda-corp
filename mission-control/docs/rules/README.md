**Read and follow every file listed here before writing or changing code.** These are the project's engineering rules (the operative form of the Pandacorp factory standards), split by purpose and technology. Only the rules that apply to this project's stack (Next.js + React + Tailwind + TypeScript, web UI, internal tool) are present.

| File | Why it's here | Covers |
|---|---|---|
| `code-conventions.md` | always | Language, naming, strict typing, constants, imports, handlers, commits, env vars, deps |
| `clean-code.md` | always | File/function size, complexity, SRP/SoC, DRY, dead code, purity, module boundaries, AI-legibility |
| `project-structure.md` | always | Source root, `core/`+`modules/`, single/multi-file folders, **tests in `_tests/`**, feature-first, promotion |
| `quality-and-testing.md` | always | Green gates, TDD, risk-based coverage, test discipline, where tests live |
| `documentation-and-decisions.md` | always | Canonical doc + decision-log discipline |
| `ai-implementation.md` | always | AI implementation discipline — comments, no in-src docs, no dead code, grounded APIs, lesson citations |
| `error-handling.md` | always | Expected vs unexpected taxonomy, typed results, never swallow, honest failure |
| `resilience.md` | always | Timeouts everywhere, single-layer retries, honest fallbacks, bulkheads, background jobs |
| `typescript.md` | TypeScript | `import type`, `satisfies`, discriminated unions, no unsafe `as`, const maps, `readonly` |
| `react.md` | React | Composition, prop-drilling ≤3, list keys, derive-don't-sync, React 19 primitives |
| `nextjs.md` | Next.js | Server Components, Server Actions + authz, optimistic UI, caching/streaming, error net |
| `api-design.md` | Next.js (has `src/app/api/**` routes) | RFC 9457 error contract, REST conventions, boundary validation |
| `styling-and-ui.md` | Tailwind | `cn()`, `@theme` tokens (no arbitrary values), light/dark, responsive |
| `accessibility.md` | web UI | Semantic HTML, keyboard/focus, forms, motion, landmarks, WCAG 2.2 |
| `web-performance.md` | web UI | Core Web Vitals, no waterfalls/dedup, runtime (long tasks, layout thrash, virtualization) |
