**Read and follow every file listed here before writing or changing code.** These are the project's engineering rules (the operative form of the Pandacorp factory standards), split by purpose and technology. Only the rules that apply to this project's stack (Next.js + React + Tailwind, web UI) are present.

| File | Why it's here | Covers |
|---|---|---|
| `code-conventions.md` | always | Language, naming, strict typing, constants, imports, handlers, commits, env vars |
| `project-structure.md` | always | `core/`+`modules/`, single-file vs multi-file folders, **tests in `_tests/`**, feature-first, promotion rule |
| `quality-and-testing.md` | always | Green gates (tests/types/lint/build), TDD, risk-based coverage, where tests live |
| `documentation-and-decisions.md` | always | Canonical doc + decision-log discipline |
| `react.md` | React | Composition, prop-drilling ≤3, list keys, derive-don't-sync, no nested components |
| `nextjs.md` | Next.js | Server Components default, Server Actions + authz, optimistic UI, global error net |
| `styling-and-ui.md` | Tailwind | `cn()`, design tokens, semantic HTML + a11y, light/dark, responsive |
| `web-performance.md` | web UI | Core Web Vitals, no waterfalls, dedup, no barrel imports, dynamic import |
