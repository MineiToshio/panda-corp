## 2026-06-16 — FRD-18: Dashboard ("Inicio") as the landing command center
**What:** New **FRD-18**. Mission Control gains a **Dashboard** as its default landing view (the old behavior opened straight into a tab); the five tabs (Board/Portfolio/Achievements/Configuration/Manual) stay reachable from the nav. The dashboard is a read-only command center with five zones, top to bottom: (1) **Desde tu última visita** — a digest of *events* (work orders closed/failed, phase moves, decisions queued, lessons captured), not totals; (2) **Tu turno** — a queue of ONLY human-gate items (pending decisions, shipped-awaiting-`review-launch`, memory backlog nudge, ideas to prioritize) — `advance_pending` deliberately excluded (it lives on the Board); (3) **Pulso de la fábrica** — a small funnel + the one metric that matters (idea→shipped conversion) + the live/no-signal indicator; (4) **Construcción y cartera** — per-project cards enriched with age-in-stage, *en vivo*/*sin señal*, inline blocker reason, and shipped shown as *estable · en operación*; (5) **Tu progreso** — the honest gamification strip (FRD-09). It composes FRD-09/12/13 and surfaces the FRD-01/15/16/17 banners. Built and approved iteratively in the prototype (`view: dashboard`): a first pass, then a research-informed v2 (added the digest, enriched the cartera, reframed the pulse as a funnel, dropped `advance_pending` from the queue per owner feedback).
**Why:** The owner asked for a landing screen that helps him *decide and take the next step* — "what needs me, where do I go" — with real value, not vanity metrics, plus a touch of honest gamification. We grounded the design in web research on analogous products (Devin Desktop's agent board, Linear Inbox, PagerDuty's "your open incidents", Vercel/GitHub-Actions build status, AgentCenter, Datadog's "alert on a lack of data", Kanban aging-WIP, HITL review-queue UX, NN/g action-centric dashboards, Lean Analytics OMTM, Octalysis White-Hat). The convergent principles: lead with a human-gate queue, manufacture substance from *events* (the since-last-visit digest) not counters, show age/stuck and no-signal not just status, tie every number to a decision, stay quiet when healthy, and reward outcomes only.
**The "Desde tu última visita" marker (key design call):** the owner flagged that with an always-open app, defining "last visit" as "last page load" would reset on every refresh and show nothing. Resolved: the `visto_hasta` marker is **persisted client-local state** (survives refresh and tab close) and advances ONLY on acknowledgement ("marcar visto") or on acting on an item — a refresh never moves it, so unseen events are never lost (email-style read/unread). When caught up, the section is never empty: it falls back to a dimmed last-24h rolling window, and (always-open) should tick up live as events arrive.
**Impact:** `mission-control/docs/frds/frd-18-dashboard.md` (new); `frd-01-data-reading.md` (reads the event stream for the digest + persists the local `visto_hasta` marker, clarified as UI state not a factory/project write); `docs/prd.md` (Scope v1 + product goal 0 + FRD range → FRD-18); `mission-control/prototype/index.html` (new `dashboard` view + default landing, `digestSection()` with localStorage-backed seen marker, enriched portfolio cards, funnel pulse, `goproj`/`goidea`/`goboard`/`seen`/`seen-sim`/`seen-reset` handlers). Verified live in the preview: all zones render, navigation works, the seen-marker survives refresh, zero console errors. Spec-anchored mock data in the prototype; the Next.js app will read real events/state. Approved as a **first version** ("not 100% convinced, ship and iterate"). No Manual change needed — the dashboard is MC's own surface, not a new factory operable item (skills/agents/rules/standards), so DR-046's derived Reference is untouched.

## 2026-06-15 — FRD-17: Proposals inbox (self-learning gate + self-suggestion)
**What:** New **FRD-17**. Mission Control gains a **Proposals inbox** ("Propuestas / Crónica del gremio") — the owner-facing surface of the self-learning loop (DR-047) and the place where MC suggests improvements on its own. It aggregates four streams: candidate lessons from `factory/memory/`, promotion proposals (a recurring lesson → standard/rule/skill), prune proposals from `/pandacorp:memory review`, and **self-suggestions** MC computes locally (bottlenecks, velocity alerts, unused skill/agent, decision-rule friction, a lesson recurring across ≥2 projects, a shipped project that merits `/pandacorp:review-launch`). For each, it shows the evidence + the exact command to act — and never acts itself (read-only, like FRD-15/16). Extends FRD-14's chips with a proposals stream; honest/dismissible per FRD-09. Also a **memory-health panel** (owner's request): pending raw-note count, candidate-lesson count, and `/pandacorp:memory` last-run staleness, with a nudge command when there's a backlog — it doubles as the on-demand refine trigger, so the cadence is scheduled sweep + this reminder (NOT tied to `implement`). And a durable **promotions queue** (lessons carry a `promotion: proposed/approved/rejected` field set by `/memory review` + `learn`): the owner reviews the full list of rules-to-promote and decides whenever — via `/pandacorp:memory status` today and this page when the app is built (owner's request).
**Why:** The owner asked for two things on Mission Control: a place to approve the lessons/promotions the factory learns, and for the app to "auto-suggest" improvements the more it is used, without being asked — the human gate of DR-047 and the visible face of the loop (`docs/proposals/09-self-learning-factory.md`). Computing suggestions locally from data MC already reads (events, status, portfolio, `factory/memory/`) keeps it read-only and Claude-call-free (FRD-01).
**Impact:** `mission-control/docs/frds/frd-17-proposals-inbox.md` (new). Spec-only — Mission Control isn't built yet (the prototype will simulate it). Factory side: the loop's engine shipped in plugin v6.1.0 (`librarian` agent + `/pandacorp:memory`), DR-047. Pending: reconcile the prototype Manual catalogs (the new `memory` skill, `librarian` agent, DR-047/DR-048) and add a "Self-learning loop" Concept page (DR-046, FRD-08).
