## Findings

### F-1 (Low, accepted) — `readBoardDoc` / `discardChangeAction` trust a client-supplied base path
- **Evidence:** `src/app/board/actions/read-doc.ts:19` (`readBoardDoc(project, relPath)` → `resolveProjectPath(project)`, absolute paths returned verbatim by `src/lib/config/config.ts:resolveProjectPath`); `src/app/projects/[slug]/_actions/discard-change.ts:33` (`discardChangeAction(projectPath, id, slug)` uses client `projectPath` as the write base in `src/lib/changes/discard-change.ts`).
- **Risk:** a Server Action is a callable POST. A caller on the loopback interface can name an arbitrary base directory: read a doc-named file (`prd.md`/`frd.md`/`architecture.md`/ADR/`decision-log.md`) under any dir, or flip `status: discarded` on a `.pandacorp/inbox/changes/<id>.md` that already exists with `ready`/`draft`. Both are tightly bounded — reads are restricted to the allow-list of discovered doc filenames (`readDoc` validates `relPath` against the discovered set and uses `lstatSync` to reject symlinks), and writes only toggle one frontmatter field on a pre-existing, specifically-typed file.
- **Remediation:** accepted for now. To harden (defense-in-depth), resolve `project`/`projectPath` against the portfolio allow-list (as `/api/live` already does via `resolveProjectDir`) instead of trusting the raw argument. Not blocking: no network exposure (127.0.0.1 / trusted LAN), single operator, no privilege boundary crossed, no secret readable (only doc-named markdown).

### F-2 (Low, accepted) — `execSync` with string interpolation in git-backed readers
- **Evidence:** `src/lib/achievements/report/phaseTransitions.ts:142,161` (`git log ... -- "${relPath}"`, `git show "${sha}:${rootRelPath}"`); `src/lib/achievements/report/flowSeries.ts:115` (`git log ... -- "${relPath}"`); `src/lib/build-track/build-track.ts:542` (static command, no interpolation).
- **Risk:** the interpolated values are work-order/status filenames discovered by walking the factory filesystem and git-provided 40-hex SHAs — never HTTP request input. A filename containing shell metacharacters could in principle break out of the double-quotes, but the factory itself creates these files (`wo-NN-MMM-slug.md`), so they are trusted in this threat model.
- **Remediation:** accepted. Defense-in-depth: migrate to `execFile("git", [...args])` (arg array, no shell) — the same pattern already used in `src/lib/pendingMerge/pendingMerge.ts:47`. Not blocking: inputs are trusted, server-side only, fail-soft (`try/catch → []`).

### F-3 (Medium → accepted) — 2 moderate transitive dependency advisories
- **Evidence:** `pnpm audit`: `postcss <8.5.10` (GHSA-qx2v-qp2m-jg93, XSS via unescaped `</style>` in stringify output) via `next>postcss` (installed 8.4.31); `js-yaml <3.15.0` (GHSA-h67p-54hq-rp68, quadratic-complexity DoS via repeated merge-key aliases) via `gray-matter>js-yaml` (installed 3.14.2).
- **Risk:** both **moderate**, both transitive, both unreachable in this app's usage. postcss runs only at build time on the app's own trusted CSS (no untrusted stylesheet is ever stringified). js-yaml parses only the factory's own trusted frontmatter/status YAML (no attacker-supplied YAML), so the DoS vector has no untrusted entry point.
- **Remediation:** accepted (no Critical/High; does not block release per DR-085). Clears when `next` and `gray-matter` publish patched transitive ranges; a `pnpm.overrides` bump (`postcss>=8.5.10`, `js-yaml>=3.15.0`) is the low-risk follow-up. Recorded, not blocking.

## Cleared checks (no finding)

- **Secrets:** none in tracked code/config or git history. `.env` / `.env.*` gitignored with `!.env.example`; `.env.example` documents only the non-secret `PANDACORP_FACTORY_ROOT` / `PANDACORP_EVENTS_FILE` paths ("needs no secrets"). No `NEXT_PUBLIC_*` secret leaked to the client bundle.
- **Write surface (path traversal):** the four bounded writes — `discardIdea`, `restoreIdea`, `setFavorite` (`lib/discard/`, `lib/favorite/`), `discardChange`, `discardBacklogItem`, `snapshotGamificationLedger` — all guard the `slug`/`id` with a `path.resolve` prefix check + `lstatSync` regular-file/symlink rejection, and rewrite exactly one frontmatter field. Adversarial tests exist (`layout.guard.test`, symlink groups).
- **Injection sinks:** the single `dangerouslySetInnerHTML` (`TimelineView.tsx:649`) injects a static hardcoded CSS constant (`FRD_DETAILS_CSS`), not user data. `react-markdown` is used without `rehype-raw` (raw HTML rendered as inert text). No `eval` / `new Function`.
- **Auth:** no home-grown auth — none at all, deliberately (ADR-0001: 127.0.0.1-bound, single operator, no network exposure). Not a home-grown-auth finding.
- **Routes:** `/api/live` (SSE) and `/api/plugin-sync` are read-only, resolve project dirs via the portfolio allow-list (never trust a client path for watch targets), and `force-dynamic` (no cache leak). `/api/plugin-sync` shells `git`/`claude plugin` read-only via `execFileSync` (arg array — safe).
- **Personal data / PII-in-logs:** the app collects no personal data of third parties; the only owner-personal artifact is the gitignored `factory/gamification-ledger.json` (counters, no PII). No PII values in log/`console` call sites.
- **Security headers / CSP:** N/A for an internal 127.0.0.1 tool (rules README explicitly excludes `web-security.md` — "internal, not public"). `deploy_target: internal`, so the CSP enforce ratchet (external-only) does not apply.
- **SSRF:** no server-side `fetch` of a user-influenced URL anywhere; the app only reads the local filesystem and shells local git.
- **Agentic (ASI01–ASI10):** N/A — no agent, no LLM, no tool invocation; the app never calls Claude (core value hypothesis).
