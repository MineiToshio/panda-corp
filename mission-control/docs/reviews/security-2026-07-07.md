# Security audit — Mission Control

- **Date:** 2026-07-07
- **Scope:** whole project (Next.js 16 / React 19, App Router). DR-085 hardening step 1a/3 — construction's last step (BL-0012).
- **Threat model:** internal tool. `deploy_target: internal`, bound to `127.0.0.1`, single local operator, no external host, no hosting/TLS/CI-CD, no network egress (architecture §1, §7, §10). **Never calls Claude / no LLM or agentic component** (ADR-0001, CLAUDE.md golden rule). This context sizes every severity below.
- **Auditor:** security-auditor agent (READ-ONLY — audit + report only; fixes are the next spawn's job).
- **Relation to prior reports:** independent re-audit over `docs/reviews/security-2026-07-06.md` (and `-2026-07-05.md`). Every prior finding re-verified against current source; no new Critical/High, no regression. Findings below are the standing set.

## Verdict

**No Critical or High findings. Release NOT blocked.** All findings are Low/Medium, accepted-with-reason under the internal-tool threat model. No fixes are required for release. The two dependency advisories and the two path/exec defense-in-depth items are recorded as non-blocking hardening follow-ups. (DR-085 blocks only on Critical/High.)

## Method

- **Secrets:** pattern grep over tracked source/config + git history scoped to `mission-control/` (`.env*`, `*.pem`, `id_rsa`, AWS `AKIA…`, OpenAI `sk-…`, GitHub `ghp_…`, Slack `xox…`, PEM headers, quoted `password`/`secret`/`api_key`/`token`). `gitleaks`/`trufflehog` not installed → manual patterns. MC shares the factory `.git`.
- **Dependencies:** `pnpm audit --prod`; lockfile present (`pnpm-lock.yaml`), one package manager pinned (`packageManager: pnpm@10.12.1`), `engines.node >=20.9`. Exact dependency names verified against the registry — no typosquats/hallucinated packages.
- **OWASP Top-10 (web):** both route handlers (`/api/live`, `/api/plugin-sync`) and all Server Actions (`board/actions`) reviewed for input validation, path traversal, authz, and injection sinks (`child_process`, `dangerouslySetInnerHTML`, `eval`, `new Function`).
- **Auth:** confirmed no home-grown auth — none at all (deliberate, ADR-0001).
- **Agentic (ASI01–ASI10):** N/A — the app is not agentic and invokes no LLM/tool; no `anthropic`/`openai`/`@ai-sdk`/`langchain` dependency or call exists. It only reads the factory filesystem and renders command *text* the owner copies.

## Findings

### F-1 (Low, accepted — defense-in-depth) — board Server Actions omit boundary schema validation on `slug`
- **Evidence:** `src/app/board/actions/actions.ts` — `discardIdeaAction(slug, reason?)`, `restoreIdeaAction(slug)`, `toggleFavoriteAction(slug, favorite)` pass `slug` straight to the write layer with no Zod parse at the action boundary (per `docs/rules/nextjs.md`: "treat every Server Action as a public endpoint").
- **Risk:** a Server Action is a callable POST reachable on the loopback interface (and, via a browser, from a malicious page — DNS-rebinding class). The write is nonetheless tightly bounded: `discardIdea`/`setFavorite`/`restoreIdea` each apply a `path.resolve` prefix check (`normalizedFile.startsWith(normalizedDir + path.sep)`) plus an `lstatSync` regular-file/symlink rejection, then rewrite exactly one frontmatter field of one pre-existing `factory/ideas/*.md` card, body preserved verbatim (`src/lib/discard/discard.ts:60-83`, `src/lib/favorite/favorite.ts:47-65`). Adversarial write-isolation tests present.
- **Remediation (non-blocking):** add a Zod schema at the action boundary (`slug` = safe filename pattern) so validation is structural, not solely delegated to the write guard. Not blocking: no network exposure, single operator, no privilege boundary, no secret readable; the traversal guard already contains the blast radius.

### F-2 (Low, accepted — defense-in-depth) — `execSync` with string interpolation in git-backed readers
- **Evidence:** `src/lib/achievements/read-model/seal.ts:36` and `factorySeal.ts:77` (`git log -1 --format=%H -- ${...}`), `src/lib/achievements/report/phaseTransitions.ts:124` (`-- "${relPath}"`), `flowSeries.ts:120`, `src/lib/build-track/build-track.ts:542`. Interpolated values are trusted: static path constants, factory `portfolio.md`-derived project paths (existence-checked, `..`-escape rejected in `factorySeal.projectStatusPathspecs`), and git-provided data. Contrast the safe form already in `src/lib/pendingMerge/pendingMerge.ts:47` — `execFileSync("git", [...args])`.
- **Risk:** the interpolated inputs never come from an HTTP request — they are local factory-owned data. No injection surface today, but the shell-string pattern makes safety input-provenance-dependent rather than structural.
- **Remediation (non-blocking):** migrate the interpolating call sites to `execFile("git", [...args])` (no shell). Not blocking: inputs trusted, server-side only, each wrapped in `try/catch` (fail-soft).

### F-3 (Medium, accepted — non-blocking) — 2 moderate transitive dependency advisories
- **Evidence:** `pnpm audit --prod`: `postcss <8.5.10` (GHSA-qx2v-qp2m-jg93, XSS via unescaped `</style>` in stringify) via `next>postcss`; `js-yaml <3.15.0` (in-tree `3.14.2`, GHSA-h67p-54hq-rp68, quadratic-complexity DoS via repeated merge-key aliases) via `gray-matter>js-yaml`.
- **Risk:** both moderate, both transitive, both unreachable in this app's usage. postcss runs only at build time on the app's own trusted CSS (no untrusted stylesheet is stringified). js-yaml parses only the factory's own trusted frontmatter/status YAML (no attacker-supplied YAML), so the DoS vector has no untrusted entry point.
- **Remediation (non-blocking):** clears when `next` / `gray-matter` publish patched transitive ranges; a `pnpm.overrides` bump (`postcss>=8.5.10`, `js-yaml>=3.15.0`) is the low-risk follow-up.

## Cleared checks (no finding)

- **Secrets:** none in tracked code/config or `mission-control/`-scoped git history (AWS/OpenAI/GitHub/Slack tokens, PEM headers, quoted credential assignments — all empty; "secret(s)" matches are the Achievements-Hall feature name, not credentials). `.env` / `.env.*` gitignored with `!.env.example`; only `.env.example` is tracked and it documents solely the non-secret `PANDACORP_FACTORY_ROOT` / `PANDACORP_EVENTS_FILE` paths ("needs no secrets"). No `NEXT_PUBLIC_*` secret.
- **Dependencies:** lockfile present, single package manager pinned, `engines` declared. All 9 runtime + devDeps are exact, well-known registry packages — no typosquat/hallucinated name.
- **Write surface (path traversal):** the bounded writes — `discardIdea`, `restoreIdea`, `setFavorite` — all guard `slug` with a `path.resolve` prefix check + `lstatSync` regular-file/symlink rejection, rewriting exactly one frontmatter field (body verbatim). Adversarial + write-isolation tests present.
- **Route handlers:** `/api/live` (SSE) and `/api/plugin-sync` are read-only, `force-dynamic` (no cache leak), 200-only, no mutation. `/api/live` resolves project dirs via the portfolio allow-list (`resolveProjectDir`, `route.ts:90-101`) — never trusts a client-supplied path for `fs.watch` targets — and cleans up watchers/timers on abort. `/api/plugin-sync` shells `git` read-only via `execFileSync` (arg array, safe).
- **Injection sinks:** no `eval` / `new Function`; the single `dangerouslySetInnerHTML` (`TimelineView.tsx:649`) injects a static module-level CSS constant (`FRD_DETAILS_CSS`), not user input. `react-markdown` used WITHOUT `rehype-raw` → raw HTML in factory markdown is rendered inert.
- **SSRF:** no server-side `fetch` of a user-influenced URL anywhere; the single client `fetch` (`plugin-sync-banner.tsx:71`) hits the hardcoded same-origin constant `/api/plugin-sync`. The app only reads the local filesystem and shells local git.
- **`[slug]` project route:** resolves by matching the URL slug against the portfolio-derived `activeProjects()` list (`nameToSlug`) — never concatenates the slug into a filesystem path; no traversal.
- **Auth:** no home-grown auth (none at all, deliberate — ADR-0001: `127.0.0.1`-bound, single operator). Not a home-grown-auth finding.
- **Personal data / PII-in-logs:** no third-party personal data collected; the only owner-personal artifact is the gitignored gamification ledger (counters, no PII). All `console.*` call sites log filenames/error objects only — no PII values (grep of logger call sites vs the data model's fields: clean).
- **Security headers / CSP:** N/A for an internal `127.0.0.1` tool (`docs/rules/README.md` explicitly excludes `web-security.md` — "internal, not public"). `deploy_target: internal` → the CSP enforce ratchet (external-only, DR-085 3b) does not apply. Baseline headers via `next.config.ts headers()` remain optional defense-in-depth, not a finding. **Rate limiting:** N/A — no public/unauthenticated network endpoints.
- **Agentic (ASI01–ASI10):** N/A — no agent, no LLM, no tool invocation; the app never calls Claude (core value hypothesis, architecture §1).
