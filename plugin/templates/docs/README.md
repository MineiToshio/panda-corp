# Document templates

The base format for every recurring document a Pandacorp project generates. A skill **reads the
template and produces the doc**, so the template is an *in-band prompt*: it is the single structural
contract (frontmatter keys, required sections, the stable-ID spine). Keep them **thin** — sections
are skeletons with explicitly-optional parts, not mandatory forms. An LLM fills every section it's
given, so a fat template yields padded, low-signal docs (DR-077).

**Agent-created docs** (the bold ones) are produced mid-`implement` by a build agent, not a phase
skill — so the **agent's prompt** (`plugin/agents/<agent>.md`) names the template path. **Docs with no
automated creator** (decision-log entries, `docs/proposals/`, `design-tokens.json`) and the
**intentionally-freeform** docs are covered in the top-level [`../README.md`](../README.md) ("How a
template is honored" + "Intentionally NOT templated").

The decision-log **entry** format is in `../shared/docs/decision-log.md.tpl`; the `.pandacorp/`
overlay templates (`guide`, `status`, `README`, `iteration`, `CLAUDE`/`AGENTS`) are in
`../shared/`. Structure & IDs: `factory/standards/structure.md` (DR-049).

## Versioning & migration policy (DR-077)

- **One version axis, not three.** The template *set* is versioned collectively under the existing
  `OVERLAY_VERSION`; a doc's lineage is the `overlay_version` / `created_with` already in its
  project's `.pandacorp/status.yaml`. **There is NO per-document `template_version` stamp** — a third
  axis with no bridge re-creates the state-fracture the pipeline audit flagged, and a birth-stamp
  lies the moment the doc is edited.
- **Retrofit by REGENERATION, not migration.** To bring an old doc onto an improved template, re-run
  its owning skill against the still-true upstream (PRD/FRD/code) — the agent that wrote it re-emits
  it. Do **not** build a structural migration engine: the docs are LLM-generated (no deterministic
  common ancestor for a safe 3-way merge), and a structural patcher's safe surface is exactly the
  low-value boilerplate, not the human-authored prose it would need to preserve.
- **Reserve in-place edits** for the rare doc carrying irreplaceable hand-edits (owner change-cards,
  hand-tuned EARS ACs); even then prefer a flag-for-review note over an auto-rewrite.
- **Drift is surfaced by a lint, not a stamp.** `.pandacorp/doc-lint.sh` reports docs whose required
  frontmatter keys are missing or whose `REQ→AC→WO` stable-ID spine doesn't resolve (run from
  `verify.sh`). It is **advisory** — it prints findings and never fails the gate, so it can't
  red-lock an existing/adopted project with a partial spine (the DR-074/075 retrofit lesson);
  promotion to a fail-closed gate is a future per-project opt-in once the spine is conformant. That,
  plus the same-turn "update canonical doc + decision log" discipline, is what keeps docs honest.

Log every template change in `plugin/docs/decision-log.md` and bump the plugin version (DR-034).
