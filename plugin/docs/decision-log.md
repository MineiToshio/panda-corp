## 2026-06-26 вҖ” doc-lint advisory gate: EXIT-trap so it can never flaky-RED the verify gate В· v9.12.2
**What:** `plugin/templates/shared/.pandacorp/doc-lint.sh` now installs `trap 'exit 0' EXIT` right after `set -euo pipefail`, enforcing its documented "ADVISORY вҖ” ALWAYS exits 0" contract no matter how the process terminates. Mirrored verbatim into the live project copy `mission-control/.pandacorp/doc-lint.sh` (kept byte-identical вҶ’ no `/upgrade` drift).
**Why:** A `/discover` run's Stop hook RED-locked on `.pandacorp/verify.sh` even though only `factory/ideas/*.md` had changed. Root cause: `doc-lint.sh` flakily exited non-zero under output redirection вҖ” a reader that closes a pipe early (`grep -q`/`head -1`) SIGPIPEs its writer, and under `pipefail`+`inherit_errexit` that killed the script *before* its final `exit 0`. `verify.sh` invokes it without `|| true`, trusting the always-0 contract, so the flake propagated to the gate. The EXIT trap is the robust, contract-honouring fix (the script still prints every advisory finding вҖ” nothing is silenced). Reproduced the flaky non-zero under redirection, then verified green Г—10 + full `verify.sh` exit 0 (60 e2e pass).
**Impact:** `plugin/templates/shared/.pandacorp/doc-lint.sh` + `mission-control/.pandacorp/doc-lint.sh` (identical). **PATCH (v9.12.1вҶ’v9.12.2)** вҖ” a gate-robustness fix, no skill/flow/behavior change. ## 2026-06-22 вҖ” `sync`: gap classification is DIRECTIONAL вҖ” never document a bug or an unbuilt feature down into the spec (DR-081 refinement) В· v8.45.0
