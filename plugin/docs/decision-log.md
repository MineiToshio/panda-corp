## 2026-06-18 — Collision-proof dev-port allocation: central ledger, scaffold reserves, blueprint materializes launch.json · v8.18.0
**What:** Implemented DR-022's "the factory records it" — which had no actual record. **`scaffold` (new step 4c)** reserves the next free **10-wide block from base 4000** in a new central ledger `factory/ports.yaml` (gitignored, personal), records `slug → base` and writes `dev_port_base` into the project's `.pandacorp/status.yaml`. **`blueprint` (step 6)** materializes the ports once the stack/dev-command exist: app = `base+0` into **`.claude/launch.json`** (so the preview/dev server starts straight from Claude Code) + app/DB/service ports into `.env` per the block offsets (Postgres `+2`, Redis `+4`; review worktree `+1`/`+3`/`+5`); it self-heals a `dev_port_base: 0` (pre-convention project) by allocating a block then. **`status.yaml.tpl`** gains `dev_port_base: 0`. **`OVERLAY_VERSION` 8.17.0 → 8.18.0** → existing projects gain the field on auto-upgrade. **MINOR → v8.18.0** (new capability, backward-compatible).
